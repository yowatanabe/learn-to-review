# learn-to-review

> **生成AI時代に「レビューする力」を鍛えるための学習リポジトリ**

## 概要

`learn-to-review` は、**コードを書く力ではなく、レビューする力を鍛える**ことに特化した学習用リポジトリです。

生成AIの進化により、コードやインフラ構成は以前よりも簡単に作れるようになりました。
一方で、

* その実装は **本当に安全か**
* **本番環境で運用してよい設計か**
* **長期的に保守・運用できるか**

を判断する力は、依然として **人間に強く求められるスキル**です。

このリポジトリでは、コード・インフラ・アーキテクチャを
**レビュアーの視点**で評価する練習を行います。

## このリポジトリで学ぶこと

このリポジトリでは、意図的に以下のような例を扱います。

* 一見正しく動きそうだが、リスクを含むコード
* 運用を考慮していないインフラ設計
* 過剰に複雑、または単純すぎるアーキテクチャ
* 生成AIが書いた「怪しい」コードや構成

**「直すこと」ではなく、「評価すること」** に集中します。

## レビュー観点

### セキュリティ

* 入力値検証は十分か
* 認証・認可の境界は明確か
* 秘密情報の扱いは適切か
* 想定外の入力や攻撃に耐えられるか

### 信頼性・可用性

* 単一障害点（SPOF）はないか
* タイムアウトやリトライは適切か
* 障害時の挙動は設計されているか

### 運用性（Operability）

* ログ・メトリクス・トレースは十分か
* 障害時に「何が起きたか」分かるか
* ロールバック・復旧は容易か

### コスト・スケーラビリティ

* 利用量増加時にコストが跳ねないか
* 不要な常時起動リソースはないか
* 現フェーズに対して過剰な構成になっていないか

### アーキテクチャ・設計

* トレードオフが整理されているか
* チームのスキルや人数に合っているか
* 将来の変更に耐えられる構造か

## リポジトリ構成（予定）


※ 構成は学習の進行に応じて変更される可能性があります。
